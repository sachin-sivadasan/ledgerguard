## Why Revenue API?

**The Problem:** Shopify's Partner API tells you about transactions, but not payment health. You don't know if a subscription is paid, overdue, or about to churn.

**The Solution:** Revenue API processes your transaction data and gives you:

<AccordionGroup>
  <Accordion title="Real-time Payment Status">
    Know instantly if a subscription is paid for the current billing cycle. No more guessing.
  </Accordion>
  <Accordion title="Risk Classification">
    Every subscription is classified: **SAFE**, **ONE_CYCLE_MISSED**, **TWO_CYCLES_MISSED**, or **CHURNED**.
  </Accordion>
  <Accordion title="Usage Billing Status">
    Track which usage charges have been billed by Shopify and which are pending.
  </Accordion>
  <Accordion title="Batch Lookups">
    Query up to 100 subscriptions in a single request for high-performance dashboards.
  </Accordion>
</AccordionGroup>

## Base URL

All API requests are made to:

```
https://api.ledgerguard.app/v1
```

## Authentication

All requests require an API key passed in the `X-API-Key` header:

```bash
curl https://api.ledgerguard.app/v1/subscriptions/gid://shopify/AppSubscription/123 \
  -H "X-API-Key: lg_live_xxxxxxxxxxxx"
```

<Note>
  API keys are created in the [LedgerGuard Dashboard](https://app.ledgerguard.app). Each key is tied to your account and has its own rate limit.
</Note>
